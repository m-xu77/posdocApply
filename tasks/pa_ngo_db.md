中国扶贫组织生态数据库（2000–2020）
数据提取与数据库构建战略（建议稿）_张老师建议
________________________________________
一、核心定位：不要做“扶贫数字仓库”
本项目不应仅仅提取：
•	项目数量
•	金额
•	培训人数
•	基地数量
而应构建：
“中国扶贫组织生态数据库”
核心问题是：
中国扶贫过程中，哪些组织进入了基层？
它们如何参与？
使用了什么治理逻辑？
资源如何流动？
不同主体如何形成协同、分工与扩散？
因此：
数据库的核心单位（unit of analysis）不是“数字”，而是：
组织—行动—资源—空间（Organization–Action–Resource–Space）
________________________________________
二、理论框架：组织生态（organizational ecology）
建议采用：
“扶贫组织生态”框架
即：
扶贫并不是单一国家行为，
而是：
多主体进入基层空间的过程。
主要主体包括：
类型	示例
中央政府	国务院扶贫办、部委
地方政府	县乡政府
金融机构	农行、信用社、政策银行
高校	定点扶贫高校
企业	国企、民企、电商平台
社会组织	基金会、协会
国际组织	UNDP 等
社区组织	村集体、合作社
研究重点：
•	谁进入？
•	什么时候进入？
•	进入哪里？
•	用什么方式？
•	带来什么资源？
•	与谁协同？
________________________________________
三、数据库总体结构（建议）
建议建立：
五大核心表（Core Tables）
________________________________________
Table 1：Organization Table（核心）(从目录就可以整理出来)
功能：
建立中国扶贫参与主体名录。
核心字段：
字段	含义
org_id	组织唯一ID
org_name	组织名称
org_type	政府/企业/高校/NGO/金融机构
administrative_level	中央/省/市/县
sector	教育/金融/交通/医疗等
ownership	国有/民营/混合
目的：
研究：
•	哪类组织增长最快？
•	哪些部门最活跃？
•	企业何时进入？
•	NGO扩张过程？
________________________________________
Table 2：Program / Action Table （不能只是记录数字，任何的项目或者行动都要记录（数字可以留空）
功能：
记录扶贫行动。
核心字段：
字段	含义
project_id	项目ID
action_type	修路/培训/贷款/产业扶贫
governance_logic	基础设施/人力资本/金融扶贫/市场整合
target_population	农户/儿童/妇女/老人
number 数量
unit 计量单位

duration	时间
implementing_org	实施主体
start_time 开始时间
注意：
不要仅记录“项目名称”。
必须抽象成：
治理逻辑（governance logic）
________________________________________
Table 3：Geography Table （整理来自其它数据源）
功能：
建立空间结构。
核心字段：
字段	含义
province	省
county	县
poverty_county	是否贫困县
mountain_plain	山区/平原
ethnic_region	民族地区
transportation_access	交通可达性
未来可连接：
•	行政区划数据库
•	GIS
•	人口普查
•	老龄化数据
________________________________________
Table 4：Resource Table
功能：
统一资源口径。
核心字段：
字段	含义
resource_type	资金/培训/基地/设备
amount	数值
unit	元/人/个
aggregation_level	个体/项目/地区
source	中央/地方/企业
极重要：
必须解决：
统计口径不一致问题。（可以细化成总量多少元，受惠人数多少）
否则：
数据不可比较。
________________________________________
Table 5：Raw Evidence Table
功能：
保留原始文本。
核心字段：
字段	含义
raw_text	原文
source_file	来源
page_num	页码
extraction_confidence	AI置信度
reviewer	审核人
原则：
永远不要删除 raw_text。
未来AI会不断改进。
________________________________________
四、最重要的提取原则
________________________________________
原则1：组织优先（organization-first）
不要：
“看到数字就抽”。
而要：
先识别：
谁在行动。
例如：
•	哪个部委？
•	哪个高校？
•	哪个企业？
•	哪个金融机构？
________________________________________
原则2：治理逻辑优先（logic-first）
不要只记录：
“修了多少路”。
而要记录：
为什么修路。
即：
其治理逻辑：
•	基础设施整合
•	市场进入
•	人力资本
•	金融整合
•	社会治理
________________________________________
原则3：空间嵌入（spatial embedding）
必须保留：
•	省
•	县
•	贫困县
•	连片特困区
•	山区/平原
因为：
扶贫本质上是：
国家资源的空间再配置。
________________________________________
原则4：时间过程（temporal process）
重点不是：
“某年做了什么”。
而是：
组织生态如何演化。
例如：
时期	特征
2000–2010	基础设施与农村公共服务
2010–2020	精准扶贫与组织动员
2020后	乡村振兴与收缩治理
________________________________________
五、最有潜力的研究方向
________________________________________
方向1：扶贫组织生态演化
研究：
•	哪些主体崛起？
•	哪些主体退出？
•	高校/企业何时大量进入？
________________________________________
方向2：治理逻辑扩散
例如：
•	从基础设施扶贫
•	到产业扶贫
•	到电商扶贫
•	到数字治理
________________________________________
方向3：空间选择机制
研究：
为什么：
•	某些县获得大量组织进入？
•	某些地区长期边缘化？
________________________________________
方向4：扶贫与行政城市化
连接：
•	县改区
•	村改居
•	镇改街道
研究：
扶贫是否推动基层空间整合？
________________________________________
六、技术建议
________________________________________
第一阶段：
taxonomy construction（最重要）
先统一：
•	组织分类
•	项目分类
•	治理逻辑
•	资源类型
不要急于全量抽取。
________________________________________
第二阶段：
semi-automated extraction
AI + 人工校验。
________________________________________
第三阶段：
spatial linkage
连接：
•	行政区划
•	GIS
•	人口数据
•	老龄化数据
________________________________________
第四阶段：
longitudinal analysis
研究：
2000–2020组织生态变化。
________________________________________
七、最终目标
不要把项目做成：
“扶贫统计数据库”
而应做成：
“中国扶贫治理生态系统数据库”
这将不仅能研究扶贫，
还能够研究：
•	国家治理
•	组织动员
•	基层空间重构
•	福利资源配置
•	行政整合
•	乡村转型
并与：
•	城镇化
•	老龄化
•	社区变迁
形成长期连接。
